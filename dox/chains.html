The close method asks an object to shut itself down in a way that will allow it to be reopened, unlike the end method which will call the destroy method which should make the object unusable, but also devoid of all resources whereas close may still keep some resources open.

<private> destroy

Destroy is called by the end method and it is here that you should clean up after yourself. The difference between destroy and end is the end is the verb that you raise on an object to ask it to go away and destroy is where you actually do the work to clean up. Think of this as the counterpart of constructor yet not called automatically.

This uses the before chain which means the last one defined is the first one destroyed
